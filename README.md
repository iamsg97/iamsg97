## System Design &amp; Architecture Principles

- **12-Factor &amp; Clean Architecture** — strict separation of config, state, and code; SOLID applied across service boundaries.
- **Event-Driven by default** — SQS/SNS fan-out + DLQ for error recovery; 99.9% message-delivery reliability in production.
- **Security as posture, not feature** — IAM least-privilege, KMS-encrypted secrets via SSM Parameter Store, VPC isolation, OAuth 2.0 / OIDC end-to-end.
- **Observability-first** — CloudWatch dashboards and alarm-driven SLOs wired before launch, not after the incident.
